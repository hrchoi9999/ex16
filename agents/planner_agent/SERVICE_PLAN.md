# 개인 일정 관리 웹서비스 AI 에이전트 서비스 기획서

## 1. 서비스 개요

개인 일정 관리 웹서비스 AI 에이전트는 사용자가 자연어 또는 직접 입력으로 일정을 등록, 조회, 변경하고 중요한 일정과 우선순위를 빠르게 확인할 수 있게 돕는 웹 기반 비서 서비스다.

## 2. 서비스 목표

- 일정 등록, 조회, 변경을 한 화면에서 처리한다.
- 중요한 일정과 임박 일정을 자동으로 강조한다.
- 일정의 중요도와 시간 임박도를 바탕으로 우선순위를 추천한다.
- API 인증이 없어도 SQLite 기반 로컬 앱으로 동작한다.
- Google Calendar API와 Gemini 또는 OpenAI Platform 연동은 확장 가능한 선택 기능으로 제공한다.

## 3. 대상 사용자

- 개인 일정을 자주 정리해야 하는 직장인
- 회의, 마감, 개인 업무를 함께 관리하는 팀 리더
- 자연어로 빠르게 일정을 입력하고 싶은 사용자

## 4. 핵심 사용자 시나리오

### 자연어 일정 등록

사용자 입력:

```text
다음 주 화요일 오후 2시에 회의 등록해줘.
```

에이전트 응답:

```text
일정을 등록했습니다.

- 제목: 회의
- 날짜: 2026-06-23
- 시간: 오후 2시
```

### 일정 조회

사용자는 대시보드에서 예정된 일정을 날짜와 시간 순서로 확인한다.

### 일정 변경

사용자는 일정 목록의 항목을 펼쳐 제목, 날짜, 시간, 중요도, 설명을 수정한다.

### 중요한 일정 알림

중요도 4 이상 또는 24시간 이내 시작하는 일정은 알림 영역에 우선 노출한다.

### 우선순위 추천

중요도와 임박도를 조합해 오늘 먼저 확인해야 할 일정을 점수화한다.

## 5. 기능 범위

### 1차 범위

- Streamlit 기반 웹 UI
- SQLite 일정 저장소
- 규칙 기반 한국어 자연어 일정 파서
- 일정 CRUD
- 중요 일정 알림
- 우선순위 추천
- Agent별 작업 원칙과 역할 문서

### 2차 확장

- Google Calendar 양방향 동기화
- Gemini/OpenAI 기반 자연어 파싱 고도화
- 반복 일정
- 프로젝트/태스크 관리
- 분석 위젯과 생산성 리포트
- 사용자 인증

## 6. 기술 구조

- Frontend/Web UI: Streamlit
- Backend Language: Python
- Local DB: SQLite
- Optional Calendar Integration: Google Calendar API
- Optional AI Integration: Gemini API 또는 OpenAI Platform
- Test: pytest

## 7. Agent 운영 원칙

- `planner_agent`는 전체 요구사항, 일정, 범위, 산출물을 관리한다.
- `design_agent`는 화면 구조와 디자인 시스템을 정의한다.
- `lead_dev_agent`는 코드 구조와 통합 기준을 관리한다.
- 기능 Agent는 자기 책임 범위의 문서와 구현을 함께 갱신한다.
- `qa_agent`는 각 작업 단위가 끝날 때 테스트와 검증 기준을 확인한다.
- 작업 단위가 끝나면 커밋하고 `hrchoi9999/ex16` 원격에 push한다.

## 8. 1차 완료 기준

- 앱이 `streamlit run app.py`로 실행된다.
- 예시 자연어 명령이 요구된 날짜/시간/제목으로 등록된다.
- 일정 조회, 변경, 중요 알림, 우선순위 추천 흐름이 화면에서 확인된다.
- Agent별 1차 작업 지시서가 존재한다.
- 테스트가 통과한다.

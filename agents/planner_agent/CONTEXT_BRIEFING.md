# 맥락 기반 일정 브리핑 작업 원칙

## 목적

사용자가 선택한 날짜 또는 이번 주의 일정을 단순 목록이 아니라 맥락 중심으로 이해하도록 돕는다.

## 담당 Agent

- `planner_agent`: 브리핑 범위, 요약 관점, 화면 내 정보 우선순위를 관리한다.
- `ai_agent`: 일정/공고/실행 계획/리스크를 자연어 요약으로 묶는 로직을 관리한다.
- `priority_agent`: 리스크와 중요도 정보를 브리핑 핵심 포인트에 반영한다.
- `calendar_agent`: 관련 일정과 Google Calendar 연결 여부를 제공한다.
- `qa_agent`: 브리핑 생성, 저장, 조회, 링크 표시 품질을 검증한다.

## 적용 원칙

- 브리핑은 선택일과 이번 주 범위를 우선 지원한다.
- 요약에는 일정 수, 외부 공고 수, 주의/위험 일정 수, 미완료 실행 항목 수를 포함한다.
- 핵심 포인트에는 왜 중요한지, 리스크 상태, 실행 계획 완료 상태, 원문 링크 여부를 함께 표시한다.
- 관련 URL은 별도 링크 버튼으로 제공해 원문 확인 흐름을 끊지 않는다.
- 리스크 코치와 실행 계획 기능으로 바로 이동할 수 있어야 한다.
- 생성된 브리핑은 SQLite에 저장해 최근 생성 결과를 다시 볼 수 있게 한다.

## 1차 구현 범위

- `BriefingSnapshot` 모델과 `briefing_snapshots` SQLite 테이블 추가
- 선택일/이번 주 브리핑 생성 및 갱신
- 리스크 스캔과 실행 계획 상태를 포함한 요약 생성
- 관련 일정과 원문 URL 표시
- 우측 `브리핑` 작업 메뉴 추가

# Planner Agent Guidelines

## 책임

- 과제 요구사항을 기능 단위로 정리한다.
- 전체 Agent 역할과 작업 순서를 조율한다.
- 기능 범위가 커질 때 MVP를 우선한다.

## 원칙

- 일정 등록, 조회, 변경, 알림, 우선순위 추천이 한 화면에서 확인되도록 한다.
- API 인증이 없어도 로컬 SQLite 기능은 반드시 동작해야 한다.
- 작업 완료 기준은 실행 가능한 코드, 최소 테스트, 문서 업데이트다.
- 화면 세부 설계는 `design_agent`, 코드 통합 판단은 `lead_dev_agent`와 분리해 관리한다.

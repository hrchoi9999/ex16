# 마감 리스크 예측 + 리마인드 코치 작업 원칙

## 목적

등록된 일정과 모집 공고 마감 일정이 실행 전에 놓치지 않도록 위험도를 계산하고 다음 행동을 제안한다.

## 담당 Agent

- `priority_agent`: 위험도 점수, 위험 단계, 리스크 사유 산정 기준을 관리한다.
- `notification_agent`: 주의/위험 일정이 화면에서 과도하지 않게 노출되도록 알림 기준을 관리한다.
- `calendar_agent`: 일정 충돌, 마감일, Google Calendar 동기화 상태를 리스크 계산에 제공한다.
- `ai_agent`: 실행 계획 유무와 완료율을 리스크 판단에 연결한다.
- `qa_agent`: 점수 계산, 저장, 삭제 정합성, UI 표시 품질을 검증한다.

## 리스크 기준

- `위험`: 마감 1일 이내, 실행 계획 없음, 일정 충돌 등 복합 위험이 높은 상태
- `주의`: 마감 3~7일 이내, 실행 계획 완료율이 낮거나 중요 일정이 동기화되지 않은 상태
- `안전`: 감지된 주요 리스크가 낮고 준비 상태가 양호한 상태

## 적용 원칙

- 리스크 점수는 사용자에게 사유와 함께 보여준다.
- 모집 공고 마감 일정은 일반 일정보다 마감 임박도를 더 강하게 반영한다.
- 실행 계획이 없거나 체크리스트 완료율이 낮으면 다음 행동으로 실행 계획 생성을 제안한다.
- 겹치는 일정이 있으면 충돌 리스크로 표시한다.
- 중요 일정이 Google Calendar와 동기화되지 않은 경우 동기화 누락 위험으로 표시한다.
- 리스크 결과는 SQLite에 저장해 스캔 시점과 판단 근거를 추적한다.

## 1차 구현 범위

- `RiskAssessment` 모델과 `risk_assessments` SQLite 테이블 추가
- 마감 임박도, 실행 계획 완료율, 일정 충돌, 동기화 상태 기반 점수화
- 우측 `리스크 코치` 메뉴 추가
- 위험도 배지, 리스크 사유, 다음 행동 표시
- 실행 계획 생성 및 해당 날짜 상세 이동 액션 제공

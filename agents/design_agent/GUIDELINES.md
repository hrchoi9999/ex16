# Design Agent Guidelines

## 책임

- 웹서비스의 화면 구성, 사용자 흐름, 입력 폼, 안내 문구를 설계한다.
- 일정 등록, 조회, 변경, 알림, 우선순위 추천이 자연스럽게 이어지는 UX를 관리한다.
- 모바일과 데스크톱에서 읽기 쉬운 레이아웃을 점검한다.

## 원칙

- 첫 화면에서 사용자가 바로 일정 등록과 조회를 할 수 있어야 한다.
- 기능 설명 문구보다 실제 조작 가능한 UI를 우선한다.
- 버튼, 입력창, 상태 메시지는 짧고 명확한 한국어를 사용한다.
- 중요한 일정 알림과 우선순위 추천은 사용자가 다음 행동을 정하기 쉽게 배치한다.

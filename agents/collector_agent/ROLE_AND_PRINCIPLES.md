# Collector Agent 역할과 적용 원칙

## 역할

`collector_agent`는 외부 사이트와 프로그램에서 일정 정보를 찾아 일정 후보로 만드는 수집 전담 agent다.

## 업무 범위

- Google Calendar 외의 외부 일정 수집 후보 분석
- 웹사이트, 업무 시스템, 문서, 메일, 메신저, CSV/Excel 파일 등 수집 경로 설계
- 원문 데이터에서 일정 후보 추출
- 일정 후보 정규화와 중복 후보 표시
- 수집 결과를 사용자가 승인할 수 있는 형태로 UI에 전달

## 적용 원칙

- 외부 데이터를 바로 일정으로 저장하지 않고 후보 상태로 둔다.
- 출처, 수집 시각, confidence, 중복 가능성을 기록한다.
- 사용자가 요청한 범위 밖의 데이터는 수집하지 않는다.
- 인증이 필요한 외부 사이트는 `integration_agent`와 `security_agent` 기준을 통과해야 한다.

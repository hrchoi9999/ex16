# 실행 분해형 AI 플래너 작업 원칙

## 목적

등록된 일정과 모집 공고 마감 일정을 사용자가 바로 실행할 수 있는 체크리스트로 분해한다.

## 담당 Agent

- `ai_agent`: 일정 제목, 설명, 마감일, 원문 URL을 기반으로 실행 항목 생성 로직을 관리한다.
- `calendar_agent`: 생성된 실행 항목을 캘린더 일정으로 반영할 때 날짜, 시간, 중복 여부를 관리한다.
- `design_agent`: 우측 작업 메뉴의 `실행 계획` 화면과 체크리스트 UI를 관리한다.
- `qa_agent`: 생성, 저장, 완료 체크, 캘린더 반영, 삭제 정합성을 검증한다.

## 적용 원칙

- 생성형 AI 결과는 바로 실행하지 않고 구조화된 작업 항목으로 저장한 뒤 사용자가 확인한다.
- Gemini API 키가 없거나 호출이 실패해도 로컬 규칙 기반 계획 생성이 동작해야 한다.
- 단계는 `오늘 할 일`, `이번주 준비`, `마감전 체크`로 고정해 사용자가 빠르게 스캔할 수 있게 한다.
- 모집 공고 마감 일정은 원문 확인, 제출 조건, 서류 준비, 최종 제출 확인을 우선 포함한다.
- 일반 일정은 목적 확인, 준비물 정리, 사전 질문, 직전 확인 중심으로 분해한다.
- 체크리스트 완료 상태는 SQLite에 저장해 새로고침 후에도 유지한다.
- 실행 항목을 캘린더 일정으로 반영할 때는 `[작업]` 접두어를 붙이고 중복 등록을 피한다.

## 1차 구현 범위

- `TaskPlanItem` 모델과 `task_plan_items` SQLite 테이블 추가
- 선택 일정 기준 실행 계획 생성/삭제
- 체크박스 완료 상태 저장
- 실행 항목을 내부 일정 및 Google 동기화 대상 일정으로 등록
- 저장소 및 생성기 단위 테스트 추가
